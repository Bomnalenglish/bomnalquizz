📌 현재 상태: 본 가이드는 콘텐츠 등록의 전체 흐름을 설명합니다. PDF 자동 파싱 UI는 별도 작업으로 추가될 예정이며, 현재는 Claude(또는 개발자)가 원문을 받아 모의고사와 같은 패턴으로 콘텐츠 파일을 생성합니다.

1. 부교재 구조 (공통)

모든 부교재는 다음 공통 구조를 따릅니다:

교재(Book)
 └ Unit
     ├ 지문 (Passage)
     ├ ① 어휘퀴즈 (sup_<book>_u<N>_voca.html)
     ├ ② 본문암기 (sup_<book>_u<N>_memory.html)
     ├ ③ 어법·어휘 (sup_<book>_u<N>_grammar.html)
     └ ④ 총괄퀴즈 (sup_<book>_u<N>_total.html)

3. 등록 절차

  1. 1단계 원문 준비: Unit별 영어 본문 + 한국어 해석 + 핵심 어휘 리스트를 준비합니다(PDF 또는 텍스트).
  2. 2단계 Claude에 전달: 채팅창에 PDF 경로를 알려주거나 텍스트를 붙여넣어 "고1 24년 6월/23년 6월과 동일 형식으로 올림포스 독해 기본1 Unit N 생성"으로 요청합니다.
  3. 3단계 자동 생성: Claude가 모의고사와 동일한 알고리즘으로 ①~④ 파일을 만들고 사전(mock_word_dict.js)에 어휘를 추가합니다.
  4. 4단계 Unit 활성화: sup_olympos_basic1_hub.html의 해당 Unit을 available:true로 변경하고 sup_..._u<N>_picker.html에서 각 유형도 활성화.
  5. 5단계 커밋·푸시: Git에 반영하면 학생들이 바로 학습 가능.

4. 모의고사와 동일한 학습 경험

학생이 고1 24년 6월 / 23년 6월 / 올림포스 기본1 / 수능특강 어느 것을 풀어도 어휘퀴즈와 본문암기의 문제 형식·화면 구성·채점 방식·결과 확인이 동일하게 작동합니다.

  • 어휘퀴즈: 모드 선택(뜻/유의어/영영) → 4지선다 → 즉시 채점 + 한국어 뜻 힌트
  • 본문암기: 지문 선택 → 빈칸/어형/옳고그른/영영/추론 → 다음 버튼으로 진행 → 단어 뜻 표시
  • 어법·어휘: 5지선다 객관식 + 어법 오류/문맥 추론/밑줄 문제
  • 총괄퀴즈: 위 4유형 종합 채점

5. 향후 확장

아래 교재는 모두 동일 구조로 추가 가능합니다:

  • 📗 올림포스 독해 기본 2 / 영어독해 / 영문법
  • 📕 리더스뱅크 (비상)

6. 데이터 구조 (개발자용 메모)

각 퀴즈 파일은 모의고사 파일과 동일한 데이터 형태를 사용합니다:

  • 어휘퀴즈: VOCA = {"1": [{no, word, meaning, syn, def}, ...]}
  • 본문암기: PASSAGES = [{no, topic, fullEn, fullKo, questions:[{type, opts, ans, meaning_hint}]}]
  • 어법·어휘: 기존 ei_lesson3_grammar.html 또는 mock_go2_2024_06_grammar.html 패턴 사용
  • 총괄: mock_go1_2025_06_total.html 패턴 (5지선다 종합)
